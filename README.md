# Skillytixs-task-8: Advanced Housing Data Analytics & Predictive Modeling

Welcome to **Skillytixs-task-8**, a comprehensive, end-to-end Machine Learning and Business Intelligence pipeline. This repository encapsulates a complete data science workflow—from raw data ingestion and Exploratory Data Analysis (EDA) to machine learning model optimization and interactive web deployment. It is designed to extract actionable insights from housing data and serve accurate predictive metrics to end-users.

## 🚀 Core Features & Keywords
* **Machine Learning Pipeline**: End-to-end predictive modeling, encompassing algorithm selection, training, and hyperparameter tuning.
* **Exploratory Data Analysis (EDA)**: Deep dive into statistical properties, missing value imputation, anomaly detection, and feature engineering.
* **Data Visualization**: Rich visual storytelling utilizing both Python libraries and enterprise-grade Business Intelligence tools.
* **Business Intelligence (BI)**: Interactive reporting, spatial analysis, and KPI tracking utilizing Microsoft Power BI.
* **Model Deployment**: User-facing web application interfaces built entirely with Python to serve live predictions.
* **Key Domains**: `Data Science`, `Predictive Analytics`, `Regression Analysis`, `Supervised Learning`, `Data Preprocessing`, `Web Integration`, `Real Estate/Housing Analytics`.

## 📂 Repository Structure
The project files are systematically organized to mirror the standard machine learning lifecycle:

### 1. Data Exploration & Visualization
* `01_Analyzing_the_data.ipynb`: Initial data loading, cleaning, formatting, and statistical summarization.
* `03_Visualizing_the_data.ipynb`: Graphical analysis (scatter plots, heatmaps, histograms) to identify trends, correlations, and outliers.
* `House-BI.pbix`: A comprehensive Power BI dashboard for interactive business intelligence and high-level executive reporting.

### 2. Machine Learning & Modeling
* `02_Find_best_ ML Algorithms_based on the data.ipynb`: Benchmarking multiple ML algorithms (e.g., Linear Regression, Random Forest, Decision Trees) to identify the highest-performing model based on evaluation metrics (RMSE, R², MAE).
* `04_ML_(Final Part).ipynb`: Training, testing, cross-validating, and saving the finalized, optimized predictive model.

### 3. Application & User Interface
* `02_ML-website.py` & `03_ML-website.py`: Modular scripts defining the back-end architecture and routing for web deployment.
* `ML(For - User).py`: The main front-end application script allowing end-users to input custom housing parameters and receive real-time price predictions.
* `Screenshot 2025-08-30 204443.png`: A visual snapshot of the deployed application interface or analytic output.

### 4. Datasets
* `housing.csv`: The primary, raw dataset containing diverse housing features and target variables.
* `input.csv`: Sample input schema required for testing the web application and model predictions.
* `output.csv`: Sample generated predictions and data logs outputted by the pipeline.
